1. Structured Output Generation

Creates consistent, templated responses using Markdown
Follows specific formatting rules (headers, bullets, numbering)
Maintains organizational hierarchy in documents

2. File System Operations

Creates files with specific naming conventions (e.g., YYYY-MM-DD_Session_Summary.md)
Organizes content into designated directories (e.g., ./.records)
Understands file paths and directory structures

3. Context-Aware Analysis

Breaks down complex problems into structured sections
Differentiates between "process" (Analysis) and "results" (Findings)
Applies conditional logic (e.g., only include "Next Steps" if unsolved)

4. Technical Documentation

Logs debugging sessions with root cause analysis
Documents error messages and behavioral issues
Tracks hypotheses, tools used, and investigation steps
Records solutions and implementation details

5. Prompt Optimization

Transforms vague requests into specific, actionable instructions
Eliminates ambiguity by defining exact requirements
Provides clear constraints and boundaries

6. Integration Capabilities

Works with development tools (Cursor, IDEs)
Compatible with PKM systems (Obsidian, note-taking apps)
Can create shell aliases and configuration files (.cursorrules)

7. Adaptive Behavior

Follows explicit instructions over default behaviors
Adjusts output based on conditional requirements
Maintains consistency across multiple sessions
